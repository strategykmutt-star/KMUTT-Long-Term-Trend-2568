## 1. ภาพรวม

Google Sheets workbook นี้คือ**แหล่งข้อมูลหลัก** (single source of truth) ของ KMUTT Long-Term Trends Dashboard ทุกตัวเลขและข้อความที่แสดงบน dashboard ล้วนมาจาก Sheets ทั้งสิ้น ไม่มีฐานข้อมูลแยกต่างหาก ดังนั้นการแก้ไขข้อมูลทำที่ Sheets ที่เดียว

**Dashboard คืออะไร?**
Dashboard คือเว็บไซต์สาธารณะที่แสดงแนวโน้มระยะยาวของมหาวิทยาลัย เข้าได้ที่:

> `<DASHBOARD_URL>`

เว็บไซต์นี้โฮสต์อยู่บน GitHub Pages ระบบจะอัปเดตข้อมูลก็ต่อเมื่อคุณกดปุ่ม **Publish** เท่านั้น การแก้ตัวเลขใน Sheets เพียงอย่างเดียวยังไม่ทำให้ dashboard เปลี่ยน จนกว่าจะ Publish

**ขั้นตอนโดยรวมคือ:**

1. เปิด Google Sheets → แก้หรือเพิ่มข้อมูล
2. กด **Check what will change (dry-run)** เพื่อตรวจสอบก่อน
3. กด **Publish all changes** เพื่ออัปเดต dashboard จริง

ง่ายมาก ไม่ต้องแตะ Git ไม่ต้องใช้ command line เลย


### Login บัญชี Google

- ใช้บัญชี Google ที่ผูกกับอีเมล **@kmutt.ac.th** หรือบัญชีที่ทีม dev กำหนดสิทธิ์ให้

## 3. โครงสร้าง Workbook

เมื่อเปิด Sheets คุณจะเห็น tab หลายอันที่ด้านล่างจอ โครงสร้างมีดังนี้:

### แท็บหลัก

| แท็บ | สีพื้นหลัง | หน้าที่ |
|------|-----------|---------|
| **📋 INDEX** | ขาว | รายการกราฟทั้งหมด 20 กราฟ พร้อมลิงก์คลิกได้ |
| **🎨 STYLE-charts** | เทา | กำหนดประเภทกราฟ, สี, หมวดหมู่ — **อย่าแตะ** |
| **🎨 STYLE-series** | เทา | กำหนด series แต่ละเส้น — **อย่าแตะ** |
| **EDU-xxx** | เหลืองอ่อน | กราฟหมวด Education (การศึกษา) |
| **PER-xxx** | เขียวอ่อน | กราฟหมวด Personnel (บุคลากร) |
| **RES-xxx** | น้ำเงินอ่อน | กราฟหมวด Research (วิจัย) |
| **FIN-xxx** | ส้มอ่อน | กราฟหมวด Finance (การเงิน) |

### วิธีใช้ 📋 INDEX

ทุกครั้งที่เริ่มงาน **เปิดที่ INDEX ก่อนเสมอ** จะเห็นตารางกราฟทั้งหมดพร้อม Chart ID และชื่อกราฟ คลิกลิงก์ในคอลัมน์ "Tab" เพื่อข้ามไปยัง chart tab ที่ต้องการได้ทันที ไม่ต้องเลื่อนหา tab เอง

### โครงสร้างภายใน Chart Tab

Chart tab แต่ละอันมีรูปแบบเหมือนกันทุกอัน แบ่งเป็น 2 โซน:

**โซนบน (แถว 1–17) — ส่วน metadata:**

| แถว | เนื้อหา | แก้ได้? |
|-----|---------|---------|
| 1 | 🔒 Chart ID | ❌ ล็อก |
| 3 | TITLE TH (ชื่อกราฟภาษาไทย) | ✅ แก้ได้ |
| 4 | TITLE EN (ชื่อกราฟภาษาอังกฤษ) | ✅ แก้ได้ |
| 5 | SUBTITLE TH (คำบรรยายภาษาไทย) | ✅ แก้ได้ |
| 6 | SUBTITLE EN (คำบรรยายภาษาอังกฤษ) | ✅ แก้ได้ |
| 8 | METHODOLOGY TH (วิธีนับภาษาไทย) | ✅ แก้ได้ |
| 9 | METHODOLOGY EN (วิธีนับภาษาอังกฤษ) | ✅ แก้ได้ |
| 10 | SOURCE TH (แหล่งที่มาภาษาไทย) | ✅ แก้ได้ |
| 11 | SOURCE EN (แหล่งที่มาภาษาอังกฤษ) | ✅ แก้ได้ |
| 13 | — data table ↓ — (divider) | ❌ ล็อก |
| 14 | series_key (ชื่อคอลัมน์ภายใน) | ❌ ล็อก |
| 15 | Name TH (ชื่อ legend ภาษาไทย) | ✅ แก้ได้ |
| 16 | Name EN (ชื่อ legend ภาษาอังกฤษ) | ✅ แก้ได้ |
| 17 | "Year (พ.ศ.)" header | ❌ ล็อก |

**โซนล่าง (แถว 18 เป็นต้นไป) — ตารางข้อมูล:**

- **คอลัมน์ A:** ปี พ.ศ. (ตัวเลขจำนวนเต็ม เช่น 2560, หรือช่วง เช่น 2560-2561)
- **คอลัมน์ B เป็นต้นไป:** ตัวเลขข้อมูลแต่ละ series (เว้นว่างได้ถ้าไม่มีข้อมูลจริงๆ)

> **หมายเหตุ:** เซลล์สีเทา = ล็อก ห้ามแก้ เซลล์สีขาวหรือสีอ่อน = แก้ได้

## 4. งานปกติ: เพิ่มข้อมูลปีใหม่

นี่คืองานที่ทำบ่อยที่สุด ปีละ 1-2 ครั้ง ทำตามขั้นตอนนี้ได้เลย:

### ขั้นตอนที่ 1 — เปิด INDEX และเลือกกราฟ

1. คลิก tab **📋 INDEX** ที่ด้านล่างจอ
2. ดูรายการกราฟทั้งหมด หาชื่อกราฟที่ต้องการอัปเดต
3. คลิกลิงก์ในคอลัมน์ "Tab" ของกราฟนั้น → จะข้ามไป chart tab ทันที

### ขั้นตอนที่ 2 — ไปที่ตารางข้อมูล

1. เมื่ออยู่ใน chart tab แล้ว เลื่อนลงมาดูแถวล่าง
2. หาแถวสุดท้ายที่มีตัวเลข (มักอยู่แถว 30–40 ขึ้นอยู่กับจำนวนปีที่มีข้อมูล)
3. คลิกเซลล์ **A ในแถวถัดไปที่ว่างอยู่** (ต่อจากปีล่าสุด)

### ขั้นตอนที่ 3 — พิมพ์ปี

- พิมพ์ **ปี พ.ศ.** เป็นตัวเลขจำนวนเต็ม เช่น `2567`
- ถ้าเป็นข้อมูลที่ครอบคลุมหลายปี พิมพ์เป็นช่วง เช่น `2566-2567`
- ระบบรับเฉพาะปี **2500–2700** เท่านั้น ถ้าพิมพ์นอกช่วงจะเกิด error ตอน Publish

### ขั้นตอนที่ 4 — พิมพ์ตัวเลข

- กด **Tab** หรือ **→** เพื่อไปคอลัมน์ B แล้วพิมพ์ตัวเลขของ series แรก
- กด Tab ต่อไปเพื่อกรอก series ถัดไป (C, D, ...)
- **ถ้าไม่มีข้อมูลของ series ใด** ให้เว้นว่างไว้ — ไม่ต้องพิมพ์ 0 (ยกเว้นถ้า 0 มีความหมายจริงๆ เช่น ไม่มีผู้สำเร็จการศึกษาเลย)
- อย่าพิมพ์เครื่องหมาย comma (,) ในตัวเลข — พิมพ์แค่ `12345` ไม่ใช่ `12,345`

### ขั้นตอนที่ 5 — ทำซ้ำกับกราฟอื่น

ทำขั้นตอน 1–4 ซ้ำกับทุก chart tab ที่ต้องการอัปเดต ไม่จำเป็นต้อง Publish ทีละกราฟ แก้ทั้งหมดให้เสร็จก่อน แล้วค่อย Publish ครั้งเดียว ระบบจะบันทึกทุกการเปลี่ยนแปลงพร้อมกัน

### ขั้นตอนที่ 6 — Dry-run แล้ว Publish

1. คลิกเมนู **`📤 Publish to Dashboard`** ที่แถบเมนูด้านบน
2. เลือก **"Check what will change (dry-run)"** — รอดูผล ถ้าไม่มี error ค่อยไปขั้นถัดไป
3. เลือก **"Publish all changes"** — รอประมาณ 1–2 นาที
4. เมื่อ modal แสดง "สำเร็จ" dashboard จะอัปเดตแล้ว

## 5. งานบ่อยพอควร: แก้ข้อมูลย้อนหลัง และข้อความ

### แก้ตัวเลขที่กรอกผิด

ไม่ต้องตกใจ แก้ได้ง่ายมาก:

1. เปิด chart tab ที่ต้องการ (ผ่าน INDEX)
2. หาแถวที่มีปีที่ต้องการแก้ (คอลัมน์ A)
3. คลิกเซลล์ที่มีตัวเลขผิด → พิมพ์ตัวเลขใหม่ทับลงไปตรงๆ → กด Enter
4. Dry-run แล้ว Publish ตามปกติ

ระบบจะตรวจจับว่าตัวเลขเปลี่ยนไปและแสดงใน diff ตอน dry-run เพื่อให้คุณยืนยันก่อน Publish จริง

### แก้ข้อความ Title / Subtitle

ชื่อกราฟและคำบรรยายอยู่ในแถว 3–6 ของ chart tab แต่ละอัน:

1. คลิกเซลล์ใน **คอลัมน์ B** ของแถว 3 (TITLE TH) หรือแถวอื่นที่ต้องการ
2. พิมพ์ข้อความใหม่ทับ → กด Enter
3. ทำเช่นเดียวกันกับเวอร์ชัน EN ถ้าต้องการอัปเดตด้วย

> **สำคัญ:** แถว 3–6 และ 8–11 ต้องมีข้อความ **ห้ามเว้นว่าง** ถ้าเว้นว่างจะเกิด error ตอน Publish

### แก้ Methodology / Source

วิธีนับและแหล่งที่มาอยู่ในแถว 8–11 ซึ่งแสดงบน dashboard เวลาผู้ใช้คลิกดูรายละเอียดกราฟ:

- **แถว 8:** METHODOLOGY TH — อธิบายวิธีนับหรือนิยามข้อมูลภาษาไทย เช่น "นับจากจำนวนนักศึกษาที่ขึ้นทะเบียน ณ วันที่ 1 พฤษภาคม"
- **แถว 9:** METHODOLOGY EN — เวอร์ชันภาษาอังกฤษ
- **แถว 10:** SOURCE TH — แหล่งที่มาภาษาไทย เช่น "สำนักงานการทะเบียน มจธ."
- **แถว 11:** SOURCE EN — เวอร์ชันภาษาอังกฤษ

แก้โดยคลิกเซลล์ใน **คอลัมน์ B** ของแถวนั้นๆ แล้วพิมพ์ทับ

### แก้ชื่อ Legend (Name TH / Name EN)

ชื่อที่แสดงบน legend ของกราฟ (เช่น "ปริญญาตรี", "ปริญญาโท") อยู่ที่แถว 15–16:

- **แถว 15:** ชื่อ legend ภาษาไทย
- **แถว 16:** ชื่อ legend ภาษาอังกฤษ

แต่ละคอลัมน์ (B, C, D, ...) ตรงกับ series แต่ละเส้นในกราฟ คอลัมน์ B ของแถว 15 = ชื่อ series แรก คอลัมน์ C ของแถว 15 = ชื่อ series ที่สอง เป็นต้น แก้ได้โดยคลิกแล้วพิมพ์ทับ

## 6. เมนู `📤 Publish to Dashboard`

เมนูนี้คือหัวใจของระบบ มี 2 ตัวเลือก:

### ตัวเลือกที่ 1 — Check what will change (dry-run)

**คลิกได้ทุกเมื่อ ปลอดภัย 100%** ไม่มีการเปลี่ยนแปลงข้อมูล dashboard จริงใดๆ เหมาะสำหรับตรวจสอบก่อนทุกครั้ง

สิ่งที่ระบบทำเมื่อคลิก:
- อ่านข้อมูลทั้งหมดจาก Sheets
- ตรวจสอบ validation ทั้งหมด (ปีซ้ำ, ปีนอกช่วง, เซลล์บังคับว่าง ฯลฯ)
- เปรียบเทียบกับข้อมูลที่ dashboard ใช้อยู่ แล้วแสดงรายการว่าอะไรจะเปลี่ยน

ถ้าไม่มี error: จะเห็นรายการ diff (ข้อมูลที่เพิ่มหรือแก้) — ตรวจว่าตรงกับที่ตั้งใจไหม แล้วค่อย Publish
ถ้ามี error: จะเห็นรายละเอียด error พร้อมแท็บและเซลล์ที่ต้องแก้ไข → แก้ก่อน แล้วค่อย dry-run ใหม่

`<screenshot of dry-run modal>`

### ตัวเลือกที่ 2 — Publish all changes

**ใช้หลังจาก dry-run ผ่านแล้วเท่านั้น**

สิ่งที่ระบบทำ:

1. ตรวจสอบ validation อีกครั้ง (ป้องกันกรณีมีการแก้ระหว่างรอ)
2. เขียนไฟล์ข้อมูล JSON จากข้อมูลใน Sheets
3. Commit และ push ไปยัง GitHub repository
4. GitHub Pages build เว็บไซต์ใหม่ (ใช้เวลาประมาณ 1 นาที)

ระหว่างรอจะเห็น modal แสดง progress แบบ live:

```
กำลังอ่านข้อมูลจาก Sheets...
กำลังตรวจสอบความถูกต้อง...
กำลัง commit ไปยัง GitHub...
กำลัง deploy...
✅ สำเร็จ — dashboard อัปเดตแล้ว
```

**ใช้เวลาทั้งหมดประมาณ 1–2 นาที** อย่าปิด Sheets หรือปิด browser ระหว่างรอ

`<screenshot of publish progress modal>`

### เคล็ดลับ: ทำ Dry-run ก่อนเสมอ

แนะนำให้ทำ dry-run **ก่อนทุกครั้ง** ที่จะ Publish จริง เพราะ:
- ถ้ามี error จะได้แก้ก่อน ไม่ต้องรอ Publish ล้มเหลว
- เห็นชัดว่าข้อมูลที่จะเปลี่ยนตรงกับที่ตั้งใจไหม
- ใช้เวลาน้อยกว่า Publish จริง

## 7. การอ่าน Error Modal

**ไม่ต้องตกใจ** ถ้า Publish หรือ dry-run แล้วเจอ error modal ระบบออกแบบมาเพื่อช่วยคุณแก้ไข ไม่ใช่แจ้งว่าคุณทำผิด error ทุกอันบอกว่า "ต้องไปแก้ที่ไหน"

### รูปแบบของ Error Modal

```
❌ พบข้อผิดพลาด กรุณาแก้ไขก่อน Publish

แท็บ: EDU-01
ฟิลด์: ปี (พ.ศ.) แถว 25
ข้อผิดพลาด: ปีซ้ำกัน — ปี 2566 มีอยู่แล้วในแถว 22

แท็บ: FIN-03
ฟิลด์: TITLE TH (แถว 3)
ข้อผิดพลาด: ค่าว่าง — ต้องระบุข้อความ
```

ให้อ่าน:
1. **แท็บ** — บอกว่าต้องไปแก้ที่ chart tab ไหน
2. **ฟิลด์/แถว** — บอกว่าต้องไปแก้เซลล์หรือแถวไหน
3. **ข้อผิดพลาด** — บอกว่าปัญหาคืออะไร

ถ้ามีหลาย error พร้อมกัน ให้แก้ทีละ error ตามลำดับ แล้ว dry-run ใหม่จนกว่าจะผ่านทั้งหมด

### กรณีที่พบบ่อย และวิธีแก้

---

#### "ปีซ้ำกัน"

**ความหมาย:** มีปีเดียวกันอยู่ในตาราง 2 แถว ระบบไม่รู้จะใช้แถวไหน

**วิธีแก้:**
1. ไปที่ chart tab ที่ระบุ
2. ดูคอลัมน์ A หาแถวที่มีปีซ้ำ (modal จะบอกแถวที่ซ้ำมาให้)
3. ถ้ากรอกผิดแถว → คลิกขวาที่เลขแถว → Delete row
4. ถ้าตั้งใจเป็น 2 ช่วงปีต่างกัน → ตรวจสอบว่าปีถูกต้องหรือเปล่า เช่น อาจพิมพ์ `2566` ซ้ำแทนที่จะเป็น `2567`

---

#### "ปีอยู่นอกช่วง 2500–2700"

**ความหมาย:** ตัวเลขในคอลัมน์ A ไม่ใช่ปี พ.ศ. ที่ถูกต้อง

**สาเหตุที่พบบ่อย:**
- พิมพ์ปี ค.ศ. แทน พ.ศ. เช่น `2024` แทน `2567`
- พิมพ์ตัวเลขผิด เช่น `25677` (พิมพ์เกิน 1 หลัก)
- วางข้อมูลจากแหล่งอื่นที่ใช้รูปแบบต่างกัน

**วิธีแก้:**
1. ไปที่แถวที่ระบุใน error
2. แก้ตัวเลขในคอลัมน์ A ให้เป็น พ.ศ. ที่ถูกต้อง (เช่น 2560–2567)

---

#### "ค่าว่างในแถวที่มีปี"

**ความหมาย:** มีปีในคอลัมน์ A แต่คอลัมน์ข้อมูล (B, C, ...) ว่างทั้งหมด หรือลืมกรอกข้อมูลหลังจากเพิ่มปีแล้ว

**วิธีแก้:**
- ถ้าตั้งใจเว้นว่าง (ไม่มีข้อมูลปีนั้นจริงๆ) → ลบทั้งแถวออก (คลิกขวาที่เลขแถว → Delete row) อย่าให้เหลือแถวที่มีแต่ปีแต่ไม่มีตัวเลข
- ถ้าลืมกรอกตัวเลข → กรอกตัวเลขให้ครบ

---

#### "ลบแถวที่ไม่ควรลบ" / "expected Year (พ.ศ.) header at A17"

**ความหมาย:** ระบบตรวจพบว่าโครงสร้างตาราง metadata เสียหาย เซลล์ A17 ต้องมีข้อความ "Year (พ.ศ.)" แต่ไม่พบ — อาจเกิดจากการลบแถวโดยไม่ตั้งใจ

**ไม่ต้องตกใจ!** นี่เป็นกรณีที่แก้ได้ง่ายมากถ้ารีบทำ:

**วิธีแก้ด่วน:**

1. กด **Ctrl+Z** (Windows) หรือ **Cmd+Z** (Mac) ทันที เพื่อ Undo การลบ
2. กด Ctrl+Z ซ้ำหลายครั้งจนกว่าแถวจะกลับมา
3. ตรวจสอบว่า A17 มีข้อความ "Year (พ.ศ.)" อีกครั้ง
4. Dry-run ใหม่

ถ้า Undo ไม่ได้แล้ว (ทำไปนานแล้วหรือปิด Sheets ไปแล้ว) → ติดต่อ dev ตามข้อ [10](#10-ติดต่อทีมพัฒนา) dev สามารถกู้คืนจาก git history ได้

---

#### "ค่าใน title/subtitle/methodology/source ว่าง"

**ความหมาย:** เซลล์ข้อความที่บังคับ (แถว 3–6, 8–11) ถูกลบหรือเว้นว่างไว้ ระบบต้องการข้อมูลครบทุกฟิลด์

**วิธีแก้:**
1. ไปที่ chart tab ที่ระบุ
2. ดูแถวที่ระบุ (เช่น แถว 3 = TITLE TH)
3. พิมพ์ข้อความที่ถูกต้องลงไปในคอลัมน์ B ของแถวนั้น
4. ต้องกรอกทั้งภาษาไทย (แถว 3, 5, 8, 10) และภาษาอังกฤษ (แถว 4, 6, 9, 11)

---

### ถ้า Error Modal ยังไม่ชัดเจน

ถ่ายภาพหน้าจอ error modal ไว้ แล้วส่งให้ dev ตามข้อ [10](#10-ติดต่อทีมพัฒนา) พร้อมบอกว่ากำลังทำอะไรอยู่ก่อนเกิด error

## 8. สิ่งที่แก้ไม่ได้ — ต้องติดต่อ Dev

บางอย่างเกินขอบเขตของ Sheets และต้องแก้ที่ระดับ code ถ้าพยายามแก้เองอาจทำให้ระบบเสียหายหรือ Publish ล้มเหลว

### ห้ามแตะ — เซลล์สีเทา (ล็อก)

เซลล์ต่อไปนี้ถูกล็อกไว้ ไม่ควรแก้แม้ว่า lock จะหลุดด้วยเหตุใดก็ตาม:

- **แถว 1** (Chart ID) — identifier ของกราฟ ถ้าเปลี่ยนระบบจะ map ข้อมูลผิด
- **แถว 13** (divider "— data table ↓ —") — sentinel ที่ parser ใช้หาตำแหน่งตาราง
- **แถว 14** (series_key headers) — ชื่อคอลัมน์ภายในที่ code อ้างถึง
- **แถว 17** ("Year (พ.ศ.)") — sentinel อีกอันที่ parser ใช้ยืนยันโครงสร้าง

### ห้ามแตะ — แท็บ 🎨 STYLE-charts และ 🎨 STYLE-series

แท็บทั้งสองนี้กำหนด:
- ประเภทกราฟ (line, bar, area ฯลฯ)
- สีของแต่ละ series (hex color code)
- ว่า series ไหนใช้เป็น KPI card highlight
- ชื่อ section และ chart type

แม้แต่การเปลี่ยนสีเล็กน้อยก็ต้องให้ dev ทำ เพราะสีถูกกำหนดเป็น hex code ที่ต้องสอดคล้องกับ design system ทั้งระบบ

### รายการงานที่ต้องให้ Dev ทำ

| งาน | เหตุผล |
|-----|--------|
| เพิ่มกราฟใหม่ | ต้องแก้ code + เพิ่ม chart tab + อัปเดต INDEX + STYLE tabs |
| ลบกราฟที่มีอยู่ | ต้องแก้ code + ลบ tab อย่างถูกวิธี |
| เปลี่ยนประเภทกราฟ (line → bar) | แก้ที่ STYLE-charts |
| เพิ่มหรือลบ series ในกราฟ | แก้ที่ STYLE-series + แถว 14–16 |
| เปลี่ยนสี series | แก้ที่ STYLE-series |
| เปลี่ยน kpi_series_key (ตัวเลข KPI หน้ากราฟ) | แก้ที่ STYLE-charts |
| เปลี่ยน URL dashboard | แก้ที่ config ของ GitHub repository |
| กู้คืนหลัง Ctrl+Z ไม่ได้แล้ว | dev ต้อง restore จาก git history |

เมื่อต้องการงานเหล่านี้ ติดต่อ dev ตามข้อ [10](#10-ติดต่อทีมพัฒนา) โดยระบุ Chart tab ที่เกี่ยวข้อง (เช่น EDU-03) และรายละเอียดที่ต้องการ

## 9. FAQ — คำถามที่พบบ่อย

### Q: ลบ tab chart ไปโดยไม่ตั้งใจ ทำยังไง?

**ไม่ต้องตกใจ** กด **Ctrl+Z** (หรือ Cmd+Z บน Mac) ทันที Google Sheets จะ Undo การลบ tab ให้ได้ หากทำไปแล้วนานหรือปิด Sheets ไปแล้ว ติดต่อ dev ทันที dev สามารถกู้คืน tab จาก git history ได้เสมอ

> **คำแนะนำ:** ถ้าไม่แน่ใจว่า tab นั้นสำคัญไหม อย่าลบ ถามก่อนดีกว่า

---

### Q: ลบแถวใน metadata โดยไม่ตั้งใจ เช่น ลบแถว 17

**Ctrl+Z ทันที** ดูวิธีแก้ทั้งหมดที่หัวข้อ [7 — "ลบแถวที่ไม่ควรลบ"](#ลบแถวที่ไม่ควรลบ--expected-year-พศ-header-at-a17)

สรุปสั้น: กด Ctrl+Z หลายครั้งจนแถวกลับมา ถ้าไม่ได้แล้ว ติดต่อ dev

---

### Q: อยากเพิ่มกราฟใหม่ เช่น กราฟจำนวนนักศึกษาต่างชาติ

การเพิ่มกราฟใหม่ต้องให้ dev ทำ เพราะต้องแก้หลายจุดพร้อมกัน ส่ง request ไปหา dev พร้อมระบุ:

- ชื่อกราฟภาษาไทยและภาษาอังกฤษ
- ประเภทกราฟที่ต้องการ (เส้น, แท่ง ฯลฯ)
- series ที่ต้องการ (เช่น รวม, แยกชาย-หญิง)
- แหล่งข้อมูล
- ข้อมูลย้อนหลังที่มี (ถ้ามี)

ควรแจ้งล่วงหน้าอย่างน้อย 1 สัปดาห์ก่อนวันที่ต้องการใช้งาน

---

### Q: Publish สำเร็จแล้ว แต่ dashboard ยังไม่อัปเดต

**ปกตินะ** บางครั้ง browser cache ทำให้เห็นหน้าเก่า ลองทำตามนี้ตามลำดับ:

1. รอ 2–3 นาที แล้ว refresh dashboard (กด **F5** หรือ **Ctrl+R**)
2. ถ้ายังไม่อัปเดต → กด **Ctrl+Shift+R** (hard refresh) บน Chrome หรือ Firefox เพื่อล้าง cache
3. ถ้ายังไม่อัปเดตหลังรอ 5 นาที → กลับไป Sheets แล้วกด "Publish all changes" อีกครั้ง (ปลอดภัย run ซ้ำได้)
4. ถ้ายังไม่ได้ → ติดต่อ dev

---

### Q: มีหลาย chart tab ที่ต้องอัปเดต ต้อง Publish ทีละอันไหม?

ไม่ต้อง แก้ทุก chart tab ที่ต้องการให้เสร็จก่อน แล้วค่อย Publish ครั้งเดียว ระบบจะ commit ทุกการเปลี่ยนแปลงพร้อมกันใน 1 ครั้ง ประหยัดเวลากว่า Publish ทีละกราฟมาก

---

### Q: กลัวว่าจะทำพัง ทำยังไงให้ปลอดภัยที่สุด?

สี่ข้อที่ช่วยได้มากที่สุด:

1. **Dry-run ก่อนเสมอ** — ถ้า dry-run ผ่าน Publish ก็จะผ่านด้วย
2. **แก้ทีละเล็กน้อย** — ไม่ต้องแก้ทีเดียวทุกกราฟ แก้ทีละ chart tab ก็ได้
3. **ดูสีพื้นหลัง** — เซลล์สีเทา = ล็อก ห้ามแก้ เซลล์สีขาว/อ่อน = แก้ได้
4. **Ctrl+Z เสมอ** ถ้าพลาด — อย่ารีบทำอย่างอื่นก่อน undo ก่อน

---

### Q: จะรู้ได้ยังไงว่า dashboard อัปเดตเมื่อกี่โมง?

ทุกครั้งที่ Publish สำเร็จ GitHub จะบันทึก timestamp ไว้ใน commit history ถ้าต้องการดูประวัติ Publish แจ้ง dev เพื่อขอลิงก์ไปหน้า commit history บน GitHub

---

### Q: เห็น error ที่ modal แต่อ่านแล้วยังไม่เข้าใจ

**ไม่ต้องเดา** ถ่ายภาพหน้าจอ error modal ไว้ก่อน แล้วส่งให้ dev พร้อมบอก:
- กำลังทำอะไรอยู่ (เพิ่มข้อมูลปีไหน แก้กราฟอะไร)
- ข้อความ error ที่เห็น (หรือแนบ screenshot)

dev จะช่วยวิเคราะห์และบอกวิธีแก้ให้

---

## 10. ติดต่อทีมพัฒนา

ถ้าพบปัญหาที่แก้เองไม่ได้ หรือต้องการให้ dev เพิ่มหรือแก้ฟีเจอร์ ติดต่อได้ที่:

**ผู้รับผิดชอบ:**

> ชื่อ: `<DEV_NAME>`
> ช่องทางติดต่อ: `<DEV_EMAIL_OR_LINE>`

### เมื่อติดต่อ ให้ระบุข้อมูลดังนี้

1. **ปัญหาคืออะไร** — อธิบายสั้นๆ ว่าเจออะไร เช่น "Publish แล้วเจอ error ปีซ้ำกัน"
2. **ทำอะไรอยู่ก่อนเจอปัญหา** — เช่น "กำลังเพิ่มข้อมูลปี 2567 ใน EDU-01"
3. **ข้อความ error (ถ้ามี)** — ถ่ายภาพหน้าจอ หรือคัดลอกข้อความ error มาด้วย
4. **Chart tab ที่เกี่ยวข้อง** — เช่น EDU-01, FIN-03

### ระดับความเร่งด่วน

| สถานการณ์ | ความเร่งด่วน |
|-----------|-------------|
| Publish ไม่ได้เลย มี error ที่แก้เองไม่ได้ | แจ้งทันที |
| ลบ tab หรือแถวสำคัญ และ Ctrl+Z ไม่ได้แล้ว | แจ้งทันที |
| Dashboard ไม่อัปเดตหลัง Publish ซ้ำหลายครั้ง | แจ้งภายในวันนั้น |
| ต้องการเพิ่มกราฟใหม่หรือเปลี่ยนโครงสร้าง | แจ้งล่วงหน้าอย่างน้อย 1 สัปดาห์ |
| คำถามทั่วไปหรือข้อสงสัย | แจ้งได้ตามสะดวก |
